How Graphic Designers Use Notion to Manage Client Work & Creative Projects

You're a designer. You create beautiful work. But managing client projects? That's chaos. Client feedback comes via email, Slack, and voice notes. Design files are scattered across Dropbox, Google Drive, and your hard drive. Deadlines slip because you lost track of when the client actually needs the final deliverable.

Notion can centralize your entire design workflow—client management, project briefs, design versions, feedback tracking, and deadline management—so you spend more time designing and less time chasing emails.

The Problem: Design Chaos

  • Feedback overload: Client leaves feedback across email, Slack, and voicemails. You miss feedback and redo work
  • Version confusion: Is this "final_v3_actualfinal.psd" the latest or is there a v4?
  • Scope creep: Client asks for "one small change" that turns into 5 revisions. No clear agreement on what's included
  • Missing deadlines: Client says they need it "by Friday" but you thought it was next Friday
  • Payment delays: You completed the design but forgot to invoice; payment is late
  • No portfolio tracking: Hard to find your best work for portfolio/case studies

The Notion Solution: 7 Key Databases

1. Client Directory

Store all client information:

  • Name, contact info (email, phone, Slack handle)
  • Company and industry
  • Budget/rate agreement
  • Preferred communication method
  • Past projects completed for them

2. Project Brief & Scope

Define each project clearly:

  • Client name and project title
  • Deliverables (logo, social media templates, website mockup, etc.)
  • Specifications (colors, dimensions, brand guidelines to follow)
  • Revision rounds included (typically 2-3)
  • Deadline and payment amount
  • Link to design brief document

3. Design Versions & Feedback

Track every iteration and feedback cycle:

  • Project name and version number (v1, v2, etc.)
  • Date submitted to client
  • Feedback received (detailed notes or link to feedback document)
  • Status: pending feedback, revision in progress, approved
  • Link to design file (Figma, PSD, etc.)

4. Timeline & Milestones

Visual project schedule:

  • Project name, milestones (brief approval, first draft, revisions, final)
  • Due dates for each milestone
  • Days until deadline (auto-calculated)
  • Status: on track, at risk, completed

5. Asset Library & File Organization

Centralized storage for project files:

  • Project name, asset type (logo files, social media templates, brand guidelines)
  • File format and location (Figma link, Dropbox path, Google Drive link)
  • Version number and last updated date
  • Notes: "Final approved version, use this for all client deliverables"

6. Invoicing & Payment Tracking

Manage project financials:

  • Client name, project, agreed rate/fee
  • Invoice date and due date
  • Payment status: invoiced, received, overdue
  • Track revision charges (if extra revisions exceed scope)

7. Portfolio & Case Studies

Document successful projects for marketing:

  • Project name, client, industry
  • Challenge (what problem did the design solve?)
  • Solution (how did you approach it?)
  • Results (client feedback, performance metrics if available)
  • Images of the final work and before/after

Real-World Workflow: Managing a Client Project

Day 1: New Project Kicks Off

A client emails: "We need a logo redesign. Budget is $2,000. Need it by April 15." You:

  • Create new row in Projects database
  • Document the brief: "Logo redesign, current logo attached, want modern + clean aesthetic"
  • Set deadline: April 15
  • Document scope: "2 revision rounds included, additional rounds $200 each"

Day 3: First Concepts Ready

You've designed 3 concepts. You:

  • Create a new row in Design Versions: "v1 - 3 concepts"
  • Upload to Figma and paste the link in Notion
  • Status: "Pending feedback"
  • Send client the Figma link: "Here are 3 logo concepts. Please review and share feedback by April 8."

Day 8: Feedback Received

Client replies: "Like concept #1 but the colors feel off. Try warmer tones." You:

  • Update Design Versions row: Add feedback to the notes field
  • Create new version: "v2 - Concept 1 revision with warm colors"
  • Status: "Revision in progress"
  • In Notion's timeline, you see you have 7 days until deadline — on track

Day 10: Revision Ready

You've revised concept #1 with warmer colors. You:

  • Update v2 status: "Ready for feedback"
  • Attach the updated Figma file
  • Send client the link: "Updated logo with warmer tones. Thoughts?"

Day 12: Client Approves

Client: "Perfect! This is the one." You:

  • Update Design Versions v2: Status = "Approved"
  • Move to Asset Library: Create "Logo Files - Final Approved" with links to all formats (PNG, SVG, AI file)
  • Add to Portfolio: Document the project with before/after, client feedback, and the final logo
  • Create invoice: $2,000 for logo redesign
  • Send to client: "Final files attached. Invoice also attached — due by April 30."

Day 22: Payment Received

Client pays. You:

  • Update invoicing row: Status = "Paid"
  • Project complete. Move to case study for your portfolio

Common Questions

Can I Embed Figma or Adobe Files in Notion?

Yes. Embed Figma prototypes directly in Notion so clients can view and comment without leaving Notion. For Adobe files, link to Dropbox or Google Drive and embed a preview.

How Do I Prevent Scope Creep?

Document the scope clearly in the Project Brief: "This project includes 2 revision rounds. Additional revisions: $X per round." Reference this in every client update. If they request out-of-scope work, update the row: "Additional request: $X, add 5 days to deadline?"

Can I Collaborate with Other Designers?

Yes. Create a shared Notion workspace. All designers see the master client list and project briefs. Each designer has their own project assignments and design versions. Comment on feedback to collaborate.
